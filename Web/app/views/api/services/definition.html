Un servicio contiene la lógica de negocio con la cual el sistema o característica opera, y por ende queda definida como un componente de negocio.

En GALE, el servicio queda separada del modelo o controlador que expone el endpoint

Restricciones

  • A diferencia del patrón MVC, donde se dispone metodos de consulta y acceso a través del modelo, en GALE, se encuentra una capa de Servicios que permite la manipulación de uno o mas modelos de acuerdo a la logica de negocio dispuesta en la API.
  • Se recomienda el Uso de Archivo DBML (Database Model), para cada uno de los controladores generados para una característica especial, y de esta forma tener un control mas limpio de la estructura del proyecto a realizar.
